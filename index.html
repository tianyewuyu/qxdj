GitHub Pages "File Not Found" 错误

您的网站已配置，但无法找到请求的文件

The site configured at this address does not contain the requested file.
If this is your site, make sure that the filename case matches the URL as well as any file permissions.
For root URLs (like http://example.com/) you must provide an index.html file.

常见原因及解决方案

1. 缺少 index.html 文件

GitHub Pages 需要根目录下有一个 index.html 文件作为默认页面。

解决方案： 确保您的仓库根目录下有 index.html 文件。

2. 文件名大小写不匹配

GitHub Pages 区分文件名大小写。如果您的链接是 About.html 但实际文件是 about.html，会导致错误。

解决方案： 检查所有链接的文件名与实际文件名的大小写是否一致。

3. 文件路径错误

HTML 中引用的文件路径可能与实际文件位置不匹配。

解决方案： 检查所有资源（CSS、JS、图片）的路径是否正确。

4. 发布分支设置错误

GitHub Pages 可能没有从正确的分支构建网站。

解决方案： 检查仓库设置中 Pages 的发布分支：

  • 前往仓库的 Settings 标签
  • 选择 Pages 选项
  • 确认 Branch 设置正确（通常是 gh-pages 或 main 分支）

5. 构建过程未完成

GitHub Pages 构建可能需要一些时间，特别是第一次部署时。

解决方案： 等待几分钟然后刷新页面。您可以在仓库的 Actions 标签中查看构建状态。

诊断工具

回答以下问题来诊断您的问题：

1. 您的仓库根目录是否有 index.html 文件？

2. 您的文件名和链接中使用的大小写是否一致？

3. 您是否设置了正确的发布分支？

基本项目结构示例

一个正确的GitHub Pages项目结构应该类似这样：

my-website/
├── index.html
├── about.html
├── css/
│   └── style.css
├── js/
│   └── script.js
└── images/
    └── logo.png
